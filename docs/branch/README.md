# docs/branch 说明

## 1. 目录定位
- `docs/branch/` 用于沉淀方法研究分支内容：问题诊断、潜在方向、论文蓝图、预研方案。
- 该目录不作为实验执行口径；实验执行仍以根目录激活文档为准。

## 2. 版本组织
- 按版本号建子目录：`docs/branch/<version>/`（例如 `v2.1`）。
- 每个版本建议至少包含：
  - `00_INDEX.md`：索引与文件说明；
  - `01_*.md`：问题诊断/优化方向报告；
  - `02_*.md`：可发论文版方法蓝图。

## 3. 迁移规则
- 分支研究成熟后，将可执行部分抽取到 `docs/proposals/<version>/`。
- 通过评审后再提升为根目录激活版，并在 `docs/archive/` 归档旧版。
