# TEN-R 主线方法与实验简报（v2.2）

版本：`v2.2`
更新时间：2026-03-10
文档定位：以论文短稿风格总结 `ds-nav` 当前主线方法与实验现状；它服务于研究沟通和论文写作，不替代根目录激活版执行文档。

## 摘要
视觉语言导航中的在线消歧常见做法是单步不确定性阈值触发扫描，但这一策略容易出现两类失稳：一类是冲突分支长期失活，导致系统只有“看一看”而不会“退一退”；另一类是触发过密，带来轨迹和时间成本爆炸。针对这一问题，`ds-nav` 当前主线提出 `TEN-R`，即一种基于时序证据趋势、触发预算与冷却约束的规则型门控框架。与 `B1` 的纯熵门控和 `Ours-R` 的 v1 阈值门控相比，`TEN-R` 不再只看单步不确定性，而是联合考虑平滑后的 `theta`、`conflict_k`、趋势项以及停滞状态，并通过 `scan_budget` 和 `cooldown_steps` 抑制误触发。现有全量结果表明，`TEN-R` 在 `val_unseen` 1839 个 episode 上同时取得更高的 `SR/SPL` 和更可接受的过程成本：`SR=0.2947`、`SPL=0.2470`、`avg_trigger_rate=0.0918`，显著优于 `B1` 与 `Ours-R`。进一步的噪声实验显示，`TEN-R` 在视觉高斯噪声增强时表现出可解释的渐进退化，而非立即崩溃。当前工作已经形成“可运行、可记录、可对比”的主线，但距离高水平论文仍有明显差距，尤其体现在理论闭环、拓扑回退实义化、完整消融与强基线对比上。

## 1. 研究背景与问题定义
视觉语言导航中的在线消歧本质上是在执行过程中动态判断：当前是否应该继续跟随基础策略，还是应该通过额外动作去降低歧义和修复偏航。`ds-nav` 的 Round007 复盘表明，早期做法主要暴露了两个核心问题：

1. `conflict` 长期接近零，导致 `BACKTRACK` 分支几乎从未真正参与决策；
2. 一旦只依赖单步阈值，就容易在保守和激进之间来回摆动，出现“触发不足”和“触发爆炸”两种极端。

因此，当前主线把问题重新表述为：在不破坏原始骨干导航器执行链路的前提下，能否构建一个具备时序感知、冲突响应和成本约束的轻量消歧门控，使其既能稳定触发纠错，又不把轨迹成本推高到不可接受的区间。

## 2. 当前主线方法：TEN-R

### 2.1 方法定位
`TEN-R` 是 `v2.0-active` 中的主线方法，方法名为 Temporal Evidence Navigation with Rule-based gating。其目标不是训练一个全新的导航器，而是在现有 backbone 外层增加一个轻量的时序证据门控层，用来决定高层动作类型：
- `FOLLOW`：直接执行 backbone 给出的原始动作；
- `LOOK_AROUND`：执行参数化环视，补充观察；
- `BACKTRACK`：执行回退动作，打断当前错误推进趋势。

### 2.2 核心信号
当前实现使用三类核心过程量：
- `theta`：融合后的不确定性信号；
- `conflict_k`：冲突信号；
- 趋势特征：`theta_slope` 与 `k_slope`。

与 v1 不同，`TEN-R` 先对原始 DS 不确定性和冲突做滞回平滑，再对冲突做饱和校准，最后叠加趋势项生成 `temporal_uncertainty` 与 `temporal_conflict`。这一步的意义在于把门控从“单步响应”升级为“趋势响应”，使系统更容易识别持续停滞和持续矛盾，而非只对瞬时尖峰做反应。

### 2.3 决策逻辑
当前 `TEN-R` 的门控由三部分组成。

第一部分是基础动作决策。系统仍保留规则门控接口，但不再只依据简单阈值，而是将 `theta`、`conflict_k` 与趋势项送入统一决策链路，得到 `FOLLOW / LOOK_AROUND / BACKTRACK` 三类动作。

第二部分是时序回退增强。当系统检测到连续停滞、较高不确定性和足够大的冲突强度共同出现时，会强制把动作提升为 `BACKTRACK`。这使得回退从“几乎不会发生”变成“在确有必要时可触发”。

第三部分是安全约束。`TEN-R` 通过 `scan_budget=0.45` 和 `cooldown_steps=10` 抑制连续触发，并额外设置 `bt_budget=0.08` 防止回退比例失控。若预算耗尽或仍处于冷却阶段，门控会自动降级为 `FOLLOW`，必要时把 `BACKTRACK` 退化为 `LOOK_AROUND`。

### 2.4 工程实现状态
从实现形态看，当前主线已经打通了三处关键插入点：
- `vln_ce_baseline/vlnce_integration/inference_hook.py`
- `vln_ce_baseline/disambig_controller.py`
- `vln_ce_baseline/vlnce_integration/action_primitives.py`

同时，step 日志也已经补齐 `theta`、`conflict_k`、`p_scan`、`p_rewind`、`budget_left`、`cooldown_left`、`backtrack_target_node` 等字段，因此可以对“为什么触发、触发后做了什么、代价是多少”给出过程级证据。

## 3. 实验设置与证据来源

### 3.1 方法对比
当前主线围绕三类可运行方法展开：
- `B1`：纯策略熵门控；
- `Ours-R`：v1 规则门控；
- `TEN-R`：当前主线。

从文档设计上还存在 `B0` 和 `TEN-L`，但就当前已闭环的有效证据而言，`TEN-L` 在 Round008 的 50 episode 快速矩阵中已经表现出“低触发 + 高成本”的不稳定模式，因此尚未进入主线；`B0` 作为官方纯 backbone 对齐组存在历史证据，但目前与 Round009 统一 evaluator 完整并排展示的主表仍以 `TEN-R / B1 / Ours-R` 为主。

### 3.2 评估维度
实验分为两层指标：
- 任务指标：`SR`、`SPL`、`NE`、`TL`；
- 过程指标：`avg_trigger_rate`、`max_trigger_rate`、`la_sum`、`bt_sum`、`bt_ratio`、`conflict_k_nonzero_count`、`action_count_sum`。

这种二层设计是当前主线的重要优点，因为它避免了只看最终成功率而忽略动作成本和过程失稳。

### 3.3 关键证据
本文档主要依赖以下轮次：
- Round008：验证 `TEN-R` 是否真的把 `conflict_k` 和 `BACKTRACK` 打开，并比较不同 `window / budget / cooldown` 配置；
- Round009：完成 `val_unseen` 全量 1839 episode 的统一主对比；
- Round010：完成 `TEN-R` 在视觉高斯噪声下的第一阶段退化探针，并加入 `B1` 弱对照。

## 4. 实验结果

### 4.1 Round008：从“能触发”到“可控触发”
Round008 的核心价值，不是追求最高任务分数，而是验证 `TEN-R` 是否真正具备结构性行为。结论是肯定的：冻结候选 `e3_ten_r_w20_budget045_cd10_temporal_fix_50ep_v6` 已经达到 `avg_trigger_rate=0.0923`、`max_trigger_rate=0.1026`、`bt_sum=403`、`bt_ratio=0.0639`、`conflict_k_max=0.2404`。这说明 `TEN-R` 不仅能触发，而且能触发回退，且没有越过预算门禁。

更重要的是，Round008 清晰暴露了主线的“稳定区”和“失稳区”：
- 当 `cooldown` 从 `10` 缩短到 `6` 时，`action_count_sum` 激增到 `13171`，说明干预频率会迅速吞噬效率；
- `TEN-L` 虽然理论上更有扩展性，但在当前实现中触发率只有 `0.044~0.046`，同时动作总量接近 `1.8w`，说明它暂时没有形成可用的收益成本平衡。

因此，Round008 的真实贡献是把“主线方法”从概念层收缩到一个可用而克制的具体实例，即 `TEN-R v6`。

### 4.2 Round009：主线全量对比
Round009 在 `val_unseen` 1839 个 episode 上给出了当前最关键的对比结果：
- `TEN-R`：`SR=0.2947`、`SPL=0.2470`、`NE=7.1512`、`TL=11.0727`、`avg_trigger_rate=0.0918`；
- `B1`：`SR=0.0587`、`SPL=0.0585`、`NE=8.1638`、`TL=1.8389`、`avg_trigger_rate=0.1218`；
- `Ours-R`：`SR=0.0185`、`SPL=0.0176`、`NE=8.7313`、`TL=0.6707`、`avg_trigger_rate=0.9631`。

这些数字说明了三点。

第一，`TEN-R` 是当前唯一同时满足“任务性能更强”和“触发率受控”的方法。它不仅优于两个对照组，而且过程上保持了 `bt_sum=18972`、`bt_ratio=0.0617`、`conflict_k_nonzero_count=306873`，证明回退机制是真实工作的。

第二，`B1` 的失败不是因为触发过多到越界，而是因为高频扫描没有转化为任务收益。它的 `action_count_sum=918560`，约为 `TEN-R` 的三倍，却只得到极低的 `SR/SPL`。

第三，`Ours-R` 的失败是典型的规则失稳。虽然它也产生大量非零冲突，但几乎每一步都在触发，`avg_trigger_rate=0.9631`，因此被过程成本和行为失控彻底吞没。

换言之，当前数据支持的结论不是“我们找到了一个更激进的消歧器”，而是“只有在时序约束和预算控制真正纳入后，消歧才可能产生净收益”。

### 4.3 Round010：鲁棒性初探
Round010 不是完整鲁棒性矩阵，而是第一阶段探针。对 `TEN-R` 而言，视觉高斯噪声强度从 `p010` 增加到 `p030` 后，任务表现呈现可解释的渐进退化。在 `p030` 下，`SR=0.2264`、`SPL=0.1745`、`NE=6.7390`、`TL=13.1291`，相对 clean subset 的变化为 `delta_SR=-0.1887`、`delta_SPL=-0.1669`。

这一结果有两个意义：
- `TEN-R` 不是“碰一点噪声就崩”，而是呈现强度相关的连续退化；
- 过程指标几乎保持稳定，说明当前退化主要来自感知信息质量下降，而不是门控本身先失控。

相比之下，`B1 p010` 在 clean subset 和噪声下几乎没有差异，本质原因不是它鲁棒，而是它在 clean 条件下就已经接近失效基线。因此，当前 Round010 更像是在建立主线的退化曲线，而非完成强竞争对照。

## 5. 当前主线的有效结论
综合 Round008-010，当前最稳妥、最不夸张的结论应是以下三条。

第一，`TEN-R` 已经完成从“概念草案”到“主线可运行系统”的转变。最关键的结构性证据是：`conflict_k` 非零、`BACKTRACK` 可触发、触发率不再爆炸，并且这些信息都有日志闭环。

第二，当前收益主要来自“把错误的高频干预压下去，同时让必要的回退真正发生”，而不是来自某个强学习模块。也就是说，`TEN-R` 的优势首先是稳定性和约束设计的优势，其次才是复杂智能。

第三，现阶段最可信的论文叙事不是“完整时序证据图已经成立”，而是“我们提出并验证了一种具备时序证据感知和成本约束的在线纠错门控原型，在当前设置下显著优于两类弱基线”。这一表述更符合现有证据边界。

## 6. 局限性与下一步
当前主线仍有四个硬伤。

一是理论与实现还有落差。文档叙事里强调结构化证据和节点级回退，但实现中 `backtrack_target_node` 仍固定为 `0`，拓扑回退还没有形成真正的“目标选择”问题。

二是核心不确定性仍是启发式混合。当前实现仍使用 `uncertainty = max(ds_uncertainty, entropy_norm)`，这说明 `TEN-R` 还不是一个完全解耦的证据推理系统，而是一个经过时序平滑和约束增强的门控系统。

三是实验矩阵不完整。当前缺少强骨干正交性、完整 E4 对照、E5-E9 消融与校准实验，因此还不能支撑更高等级的研究主张。

四是绝对性能仍不够强。虽然 `TEN-R` 在当前内部对比里是显著最优，但其绝对 `SR/SPL` 还不足以直接支撑“顶刊级实用系统”主张。

因此，若要继续推进，最有价值的下一步不是继续细扫阈值，而是：
- 把 `BACKTRACK(target_node)` 做实，完成真正的拓扑目标选择；
- 明确拆分 `U_epi / U_con / U_pol`，减少启发式 `max` 融合；
- 补齐 E4-E9，尤其是鲁棒、消融、校准和跨骨干实验；
- 在论文叙事中严格区分“当前已证实的原型贡献”和“尚未完成的完整证据图愿景”。

## 7. 结论
从研究推进阶段来看，`ds-nav` 当前主线已经越过“只是一个想法”的阶段，进入“有证据支撑的可运行方法原型”阶段。`TEN-R` 的真正价值在于，它把在线消歧从单步阈值规则推进到了时序趋势、回退触发和成本约束共同作用的框架，并在现有全量实验中给出明确优于 `B1` 和 `Ours-R` 的结果。但如果目标是顶会或顶刊，这个版本还不能被包装成终局方案；它更像是一条已经找到正确方向、但仍需补足理论与实验纵深的主线。
